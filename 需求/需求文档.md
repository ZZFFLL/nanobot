# 项目开发约束：
1、这个项目最重要的首要原则是，在能达到相对高实现度、效果好的情况下，不对原nanobot项目做侵入性的重构和改造，尽量使用钩子等手段进行注入。
2、基于SOUL模块进行改造，而不能对nanobot本体做破坏性的改造，因为，nanobot的上游项目的更新未来还需要进行合入到我的分支中的，所以不能出现严重的侵入性改造
3、在更加的功能效果实现前提下，在能达到90%的功能效果下，尽量简化实现方式，不要出现不应该的复杂化，除非有不可逾越的设计问题（唯一的实现方案）

4、所有特性功能决策，不能由程序来硬编码逻辑判断，比如情感演进需要LLM根据聊天上下文、历史信息、当前SOUL状态等方面来进行动态推进。这个负责推演的LLM可以通过相关配置文件来进行维护模型调用地址、key、温度、上下文等关键性的字段值并能动态读取此配置文件、

5、所说的由LLM动态演化指的是需要有理论依据的，这个理论依据也就是方法论，是人类总结出来的方法论，而不是单方面的方法论甚至没有。比如，性格的演化，它是基于荣格八维来作为方向、系统性的进行推演，而不是“用户我一句话说你是什么样子就是什么样子了”。

# 需求方向

## 2026年4月14日

1. 基于开源Agent二次开发，打造云端陪伴型数字伴侣，不追求绝对自主意识的数字生命
2. 定位为陪伴型云端意识体，具备自主能力，拒绝用户说什么就是什么的无底线顺从
3. 解决Agent对话机械感、无类人真实情感的两大核心痛点
4. 实现情绪、人格等类人特性的自我进化与发展
5. 类人特性由LLM基于标准化方法论规则驱动演化，而非硬编码逻辑
6. 用标准化规则框定演化范围与方向，规避无规则、不稳定的演变
7. 程序负责数值计算闭环兜底，不依赖简单Prompt实现
8. 不开发前端界面，Agent框架自主选择，方案需可落地执行

## 验收标准

### 1 核心功能验收标准

1. 核心锚点系统正常运行，任何修改核心锚点的指令都会被拒绝，不会执行；
2. 终身记忆系统正常运行，可准确调用 3 个月以上的核心记忆细节，无记忆丢失、矛盾；
3. 可控演化体系正常运行，每周自动生成复盘日志，每月自动生成校准报告，演化不突破预设范围；
4. 核心引擎成功嵌入所选 Agent 框架，所有功能正常运行，不依赖特定框架的原生能力。

### 2 陪伴体验验收标准

1. 对话自然无机械感，口语化表达，无书面化、分点式、标准答案式内容；
2. 共情能力达标，所有情绪回应都基于用户专属记忆，无空泛通用话术；
3. 主动陪伴能力达标，每日主动发起贴合场景的话题，不频繁打扰用户；

### 3 演化效果验收标准

1. 经过 1 个月的相处，核心人格与初始锚点完全一致，无根本性偏移；
2. 经过 1 个月的相处，陪伴能力、情绪表达能力有明显提升，符合定向演化方向；
3. 演化过程全链路留痕，可查看每一次复盘、校准的内容，可随时干预。
